## 機能詳細

### ① 睡眠記録・入力フロー

**データ取得の優先順位**
```
Health Connect（自動）→ データなし or 未連携 → 手動入力
```

**Health Connect取得条件（ユーザー側）**
- Android 14以降 or Health Connectアプリ導入済み
- Samsung Health・Fitbit等の睡眠記録アプリ使用中
- そのアプリのHealth Connect書き込みをONにしている
- YOAKEへの読み取り権限を許可している

**Health Connect取得条件（アプリ側）**
- Manifestにパーミッション宣言
- 実行時に権限リクエスト
- Google PlayのHealth Connect審査承認済み（リリース前に申請必須）

**初回連携時の過去データ一括インポート**
```
オンボーディング or マイページで Health Connect 連携許可
    ↓
過去90日分の SleepSession を一括フェッチ
    ↓
Firestore に保存（既存の date ドキュメントがなければ作成・重複はスキップ）
    ↓
「○日分のデータを取り込みました」と完了メッセージを表示
```

- HC にはインストール前のデータも蓄積されているため、連携した瞬間から過去データを分析・グラフ表示できる
- 書き込み数が多くなるため Firestore バッチ処理（500件単位）を使用する
- 再連携・再インポート時は既存ドキュメントを上書きせずスキップ（ユーザーの手動修正を保護）

**過去データの制約**
| 制約 | 内容 |
|---|---|
| 書き込み期間のみ取得可 | HC への書き込みが無効だった期間のデータは取れない |
| メーカーによる保持期間の差 | Fitbit は最大1年、Samsung Health は制限なし、など |
| スリープステージはデバイス依存 | 安価なバンドは睡眠時間のみ。深睡眠・REM は上位機種のみ |
| ユーザーが削除していたデータ | HC 上で消されていると取得不可 |

**手動入力項目**
- 就寝時刻・起床時刻（タイムピッカー）
- 寝つき：すぐ / 少し時間かかった / なかなか寝れなかった
- 目覚め：😊 すっきり / 😐 ふつう / 😫 つらい
- 習慣チェック（後述）
- メモ（任意・自由記述）

**記録タイミングのハンドリング**
```
朝8時にプッシュ通知（有料）
    ↓
当日23:59までに記録がない場合
    ↓
翌日アプリ起動時に「昨日の記録をつけますか？」バナー
    ↓
無視したらスキップ（強制しない）
```

**エッジケース**
| ケース | 対応 |
|---|---|
| 日付をまたいで入力（深夜2時就寝等） | 就寝日ベースで記録 |
| 昼寝データがHealth Connectに混入 | 6時間未満 & 昼間のデータは除外 |
| Health Connect未連携ユーザー | 手動入力のみで全機能動作 |
| 記録が0日 | AIひとことは「データが溜まったら分析します」と表示 |

### ② スコア計算ロジック

**基本方針**
- Health Connectありとなしで配点を切り替える
- どちらのユーザーも100点満点になるよう重みを再配分

**Health Connectあり（100点満点）**
| 項目 | 配点 | 計算方法 |
|---|---|---|
| 睡眠時間 | 30点 | 7.5〜9h→30点、7〜7.5h→25点、6〜7h→18点、5〜6h→10点、それ以外→5点 |
| 就寝時刻 | 20点 | 22〜23時→20点、0時→15点、1時→8点、それ以外→3点 |
| 深睡眠割合 | 15点 | 20%以上→15点、15%→11点、10%→7点、それ以下→3点 |
| 目覚め主観 | 15点 | 良い→15点、普通→9点、悪い→3点 |
| 睡眠連続性 | 10点 | 覚醒0回→10点、1回→8点、2回→5点、3回→2点、4回以上→0点 |
| 寝つき主観 | 10点 | 速い→10点、普通→6点、遅い→2点 |

**手動入力のみ（100点満点）**
| 項目 | 配点 |
|---|---|
| 睡眠時間 | 40点 |
| 就寝時刻 | 25点 |
| 目覚め主観 | 20点 |
| 寝つき主観 | 15点 |

**継続ボーナス・ペナルティ**
- 就寝時刻のばらつき20分未満 → +5点
- 就寝時刻のばらつき40分未満 → +2点
- 就寝時刻のばらつき90分超 → -5点
- 9時間超の睡眠 → -5点
- 最終スコアは0〜100でクランプ

**スコアラベル**
| スコア | ラベル | 色 |
|---|---|---|
| 90〜100 | 最高 | 緑 |
| 75〜89 | 良好 | 黄緑 |
| 60〜74 | 普通 | 黄 |
| 40〜59 | やや不足 | オレンジ |
| 0〜39 | 要改善 | 赤 |

**睡眠負債計算**
- 直近14日の「目標睡眠時間 − 実睡眠時間」の合計
- 各SleepLogにsleepDebtMinutesとして保存

### ③ AI分析（Claude API使用）

**モデル**：Claude Haiku 4.5（全機能）
**呼び出し経路**：Cloud Functions 経由（APIキーはサーバーサイドで管理）

#### 毎朝ひとこと（無料）
- トリガー：アプリ起動時（当日未生成の場合）
- 入力：直近14日のsleepLogs + 習慣データ + 目標 + 季節情報
- 出力：2〜3文の日本語アドバイス
- キャッシュ：Firestoreに保存・当日中は再生成しない
- Cloud Functions 経由で生成（クライアントは関数を呼ぶだけ）

**システムプロンプト**
```
あなたは睡眠改善をサポートするAIアシスタントです。
ユーザーの睡眠データを見て、毎朝ひとこと日本語でアドバイスしてください。

ルール：
・2〜3文以内で簡潔に
・ポジティブなトーンを心がける
・具体的な数字を1つ以上使う
・説教くさくならない
・毎回同じ内容にならないようにする
・習慣データと睡眠スコアの相関があれば積極的に言及する
・ユーザーの目標に対する進捗も触れる
```

#### 週次AIレポート（有料）
- トリガー：月曜日の初回アプリ起動
- 入力：直近14日のsleepLogs + 習慣サマリー + 統計
- 出力：以下の構成
  - 📊 今週の総評
  - ✅ 良かった点
  - 💡 改善できる点
  - 🎯 来週のアクション
- 条件：先週分のログが3日以上あること
- Cloud Functions 経由で生成

#### AIチャット（有料）
- 直近14日のsleepLogsをシステムコンテキストとして保持
- 会話履歴は1セッション最大50メッセージに制限
- 1日あたり30回の送信制限
- 出力：フレンドリーなタメ口調
- Cloud Functions経由でAPI呼び出し（APIキーはクライアントに置かない）

### ④ スマートアラーム（有料・ウェアラブル必須）

**方針**
- スマホ単体（加速度センサー）は精度が低いためウェアラブル連携限定
- ウェアラブルなしユーザーには通常アラームのみ提供
- 制限ではなく「ウェアラブル連携の価値」として訴求

**仕組み**
```
起床希望時刻の30分前から監視開始
    ↓
Health ConnectからリアルタイムでSleepStageをポーリング
    ↓
レム睡眠 or 覚醒に近い状態を検知 → アラーム起動
    ↓
設定時刻を超えたら問答無用で起動（フェイルセーフ）
```

**技術実装**
- Foreground Serviceで監視（バックグラウンド対応）
- AlarmManagerでフェイルセーフ（setExactAndAllowWhileIdle）
- バッテリー最適化の除外をユーザーに案内

**無料/有料分岐**
| 機能 | 無料 | 有料 |
|---|---|---|
| 通常アラーム | ✅ | ✅ |
| スマートアラーム（ウェアラブル必須） | ❌ | ✅ |
| 起床ウィンドウ調整 | ❌ 30分固定 | ✅ 10〜60分 |
| スヌーズ間隔 | ❌ 5分固定 | ✅ カスタム |
| スヌーズ回数 | ❌ 2回固定 | ✅ 5回まで |
| アラーム音選択 | ❌ 1種類 | ✅ 複数 |

**アラーム発火後**
- 「起きた！」タップで起床時刻を自動記録 → SleepLogに反映

### ⑤ 睡眠日記（習慣チェック + AI相関分析）

**入力UI（睡眠記録と同一画面・所要時間30秒以内）**

デフォルト6項目（無料）：
- ☕ カフェイン
- 🍺 飲酒
- 🏃 運動
- 📱 就寝前スマホ
- 😤 ストレス高め
- 🛁 入浴

カスタム項目（有料・上限20項目）：
- 任意の名前 + 絵文字で追加可能
- 並び替え・削除・ON/OFF切り替え

**AI相関分析**
- カスタム項目名をそのままプロンプトに含めて渡す
- サンプル数5件未満の項目は「データ収集中」として非表示
- 「あと○日で分析できる」表示で記録継続を促進

**習慣別スコア比較グラフ（有料）**
- 各習慣の「あり/なし」でスコアを集計・比較表示
- 動的集計（項目名をキーに）

### ⑥ 目標設定

**設定項目**
- 目標睡眠時間（例：7時間30分）
- 目標スコア（70点以上 / 80点以上 / 90点以上）
- 就寝目標時刻（任意・例：23:00）

**ホーム画面への反映**
- 今週の目標達成進捗バー（例：5/7日）
- 「目標まであと○分」の差分表示

**AIへの反映**
- 全プロンプトに目標を含める
- 目標に対する進捗をアドバイスに反映

## Firestore設計

### コレクション全体図
```
firestore
 ├── users/{userId}
 │    ├── profile（ドキュメント）
 │    ├── subscription（ドキュメント）
 │    ├── goal（ドキュメント）
 │    ├── sleepLogs/{date}        ← dateがドキュメントID（"2026-03-24"）
 │    ├── aiReports/{key}         ← "2026-03-24"（日次）or "2026-W12"（週次）
 │    ├── habitTemplates/{habitId}
 │    ├── bodyLogs/{date}         ← 有料
 │    └── chatHistory/{chatId}
 │
 └── appData
      └── defaultHabits（ドキュメント）
```

### データモデル

#### UserProfile
```javascript
{
  displayName: String | null,
  height: Number | null,      // cm
  weight: Number | null,      // kg
  createdAt: Timestamp,
  lastActiveAt: Timestamp
}
```

#### Subscription
```javascript
{
  plan: "free" | "monthly" | "yearly",
  status: "active" | "expired" | "trial",
  trialStartAt: Timestamp | null,
  trialEndAt: Timestamp | null,
  currentPeriodEndAt: Timestamp | null,
  trialUsed: Boolean
}
```

#### UserGoal
```javascript
{
  targetHours: Number,        // 7.5
  targetScore: Number,        // 80
  bedTimeTarget: String | null, // "23:00"
  updatedAt: Timestamp
}
```

#### SleepLog（/{date}）
```javascript
{
  date: String,               // "2026-03-24"（ドキュメントIDと同じ）
  bedTime: Timestamp,
  wakeTime: Timestamp,
  totalMinutes: Number,

  // Health Connect由来（任意）
  deepSleepMinutes: Number | null,
  remMinutes: Number | null,
  lightSleepMinutes: Number | null,
  awakenings: Number | null,
  heartRateAvg: Number | null,

  // 主観評価
  sleepOnset: "FAST" | "NORMAL" | "SLOW",
  wakeFeeling: "GOOD" | "NORMAL" | "BAD",

  // 睡眠日記
  habits: [
    {
      id: String,             // "habit_001"
      label: String,          // "瞑想した"
      emoji: String,          // "🧘"
      checked: Boolean
    }
  ],
  memo: String | null,

  // 計算結果
  score: Number,              // 保存時に計算
  sleepDebtMinutes: Number,   // この日時点の累積負債

  // メタ
  source: "HEALTH_CONNECT" | "MANUAL",
  createdAt: Timestamp,
  updatedAt: Timestamp
}
```

#### AiReport（/{key}）
```javascript
{
  type: "daily" | "weekly",
  content: String,            // AI生成テキスト
  generatedAt: Timestamp,
  inputSummary: String,       // デバッグ・再生成用
  modelUsed: String,
  tokenCount: Number | null
}
```

#### HabitTemplate（/{habitId}）
```javascript
{
  id: String,
  label: String,              // "瞑想した"
  emoji: String,              // "🧘"
  isDefault: Boolean,
  isActive: Boolean,
  order: Number,
  createdAt: Timestamp
}
```

#### appData/defaultHabits
```javascript
{
  habits: [
    { id: "default_01", label: "カフェイン",    emoji: "☕", order: 1 },
    { id: "default_02", label: "飲酒",          emoji: "🍺", order: 2 },
    { id: "default_03", label: "運動",          emoji: "🏃", order: 3 },
    { id: "default_04", label: "就寝前スマホ",  emoji: "📱", order: 4 },
    { id: "default_05", label: "ストレス高め",  emoji: "😤", order: 5 },
    { id: "default_06", label: "入浴",          emoji: "🛁", order: 6 }
  ]
}
```

### セキュリティルール
```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null
                         && request.auth.uid == userId;
    }
    match /appData/{document=**} {
      allow read: if request.auth != null;
      allow write: if false;
    }
  }
}
```

### 必要なインデックス
```
sleepLogs: date(降順) × score
sleepLogs: date(降順) × habits.checked
aiReports: type × generatedAt(降順)
```

10.【決定済み】常時起動計測（ポケモンスリープ式）は**実装しない**
  - Sleep as Android・熟睡アラームが常時起動を持つのは「スマホだけで計測」が差別化のため。Yoake とコンセプトが異なる。
  - Health Connect / HealthKit 連携でウェアラブルの高精度データを取得できるため、スマホ単体計測で精度競争しても意味がない。
  - 常時起動はバッテリー消耗・マイク使用によるプライバシー審査コスト・ユーザー説明コストが増える。
  - Yoake の健康観は「記録と振り返りによる自己理解（自己認識型）」であり、常時起動の「受動的に計測される」設計思想と真逆。

11.【決定済み】ウェアラブル未所持ユーザーはグラデーション設計で拾う（完全切り捨てはしない）
  - 日本市場でのスマートウォッチ所持率は成人の2〜3割程度（2025年時点）。切り捨てると母数が大きく絞られる。
  - 手動入力を「ちゃんと残している」設計はウェアラブルなしユーザーへのメッセージ。塞ぐとブランドの一貫性が崩れる。
  - 方針：**メインターゲット = ウェアラブルユーザー（有料プレミアム）、手動ユーザー = 無料〜ベーシックで拾う**
  - プレミアム機能は「ウェアラブルがあると嬉しいが必須ではない」設計を維持する（Health Connect の詳細分析、スマートアラームなど）。
  - 代替施策として就寝リマインダー通知（有料）で「常時起動感」をオーバーヘッドなく代替する。

12.【決定済み】就寝・起床時刻の入力精度を±30分以内に収める設計
  - 目標：手動入力でも記憶頼りにならず、実態に近い時刻を記録できるようにする。
  - **最低限実装（必須）**
    - 就寝リマインダー通知に「今から寝ます」ボタンを追加 → タップ時刻を就寝時刻として記録（就寝側の精度確保）
    - アラームの「起きた！」タップ時刻を SleepLog の起床時刻に自動セット（起床側の精度確保）
  - **追加施策（UX改善）**
    - 朝の記録促進通知を起床予定時刻の直後（5〜10分後）に飛ばす（記憶が鮮明なうちに入力させる）
    - Health Connect 連携ユーザーは HC データで時刻フィールドを先入力し「違う場合だけ修正」方式にする
    - 入力 UI を「昨日と同じ？」「30分くらい早かった？」などの相対選択肢方式にする（ゼロから時刻入力させない）
  - **ユーザータイプ別の最適経路**
    | ユーザータイプ | アプローチ |
    |---|---|
    | HC連携 + アラーム使用 | ほぼ自動（手修正のみ） |
    | アラームのみ | 就寝タップ + アラーム連携で起床を自動化 |
    | 手動のみ | 就寝タップ + 朝の早期通知 + 相対選択 UI |
