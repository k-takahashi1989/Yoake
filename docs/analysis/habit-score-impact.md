# 習慣別スコア影響 — 分析・改善メモ

作成日: 2026-03-28
対象ファイル: `src/screens/Report/components/HabitCorrelationCard.tsx` / `src/screens/Report/utils/habitStats.ts`

---

## 現状の実装

### データ構造（`HabitStat` 型）

```typescript
export interface HabitStat {
  id: string;
  label: string;
  emoji: string;
  withAvg: number;      // その習慣を実行した日の平均スコア
  withoutAvg: number;   // 実行しなかった日の平均スコア
  withCount: number;    // 実行日数
  withoutCount: number; // 未実行日数
}
```

### 表示内容

- **棒グラフ**: `withAvg` を表示。全体平均以上 → 緑、未満 → オレンジ。基準線 = avgScore
- **各行**: 絵文字 + ラベル + [実行日平均チップ] + [未実行日平均チップ] + [差分 `+N` / `-N`]
  - 差分色: `diff > 2` → 緑 / `diff < -2` → 赤 / それ以外 → グレー
- **除外条件**: `total < 3`（実行＋未実行の合計が3未満）
- **ソート**: `|withAvg - withoutAvg|` の大きい順
- **最大表示数**: 6習慣

---

## 問題点

### 1. 統計的信頼性の欠如

現在の閾値 `total < 3` は「実行2日・未実行1日」でも表示してしまう。

**問題シナリオ：**
- `withCount=3`（点数: 82, 85, 78 → 平均82）
- `withoutCount=17`（平均75）
- 表示: **+7点** ← 「たまたま体調が良かった3日間」の可能性がある

**統計的最低ライン（行動変容研究の基準より）：**

| 信頼度           | 各グループ最低件数 | 根拠                            |
|----------------|------------|-------------------------------|
| 参考値（注意書きつき） | ≥ 5件       | 単純平均の安定性                    |
| 準信頼            | ≥ 10件      | 中心極限定理の実用的下限                |
| 信頼（断言可）       | ≥ 20件      | ウィルコクソン検定が有効になる水準           |

**現状はこの基準を全く満たしていない。**

---

### 2. サンプル数が見えない

`withCount=3` と `withCount=30` が全く同じ外見で表示される。
分析好きユーザーが「これ信頼できる？」と感じて離脱するリスクがある。

---

### 3. 実行頻度コンテキストの欠如

`withCount=5/30日` と `withCount=5/7日` が同じ「5回」として見える。
「月に5回（低頻度）」と「先週5回（高頻度）」では意味が全く異なる。

---

### 4. 交絡変数問題

「運動した日 = アルコールも控えた日」が多い場合、運動の効果とアルコール回避の効果が混在する。
現状の設計では分離不可能。ユーザーがこれに気づくとデータ全体への信頼を失う。

---

### 5. 継続性・トレンドが見えない

「3ヶ月前は毎日やっていたが最近やめた習慣」でも `withAvg` が高く出る。
直近の傾向が反映されない。

---

## ユーザー層別の反応予測

| ユーザー層            | 感じる不満                                        |
|--------------------|----------------------------------------------|
| ライトユーザー          | 「+5点ってそうなんだ」で終わる。行動変容につながらない               |
| 習慣管理に敏感なユーザー | 「本当に？これ信頼できる？」→ 不信感                          |
| 分析好きユーザー        | 「交絡してるでしょ」→ アプリへの根本的不信 → **離脱リスク大** |

## 改善提案（優先順位順）

---

### 🟢 優先度A — 既存型のデータだけで実装可能（コスト：小）

#### A-1. サンプル数の可視化 ＋ 信頼度バッジ

**表示イメージ：**

```
🏃 運動  実行: 82点(12日)  未実行: 75点(8日)   +7
😤 禁酒  実行: 79点(4日)   未実行: 74点(16日)  +5  参考値
```

**信頼度の定義：** `min(withCount, withoutCount)` で判定

| min値   | 信頼度 | 差分色             |
|--------|-----|-----------------|
| < 5件  | 低  | `#555`（灰色・抑制）  |
| 5〜9件 | 中  | `#888`（薄色）      |
| ≥ 10件 | 高  | 緑/赤（現在と同じ）    |

**必要な変更：**
- `HabitCorrelationCard.tsx` の `HabitCorrelationRow` で `minCount` を計算して色・バッジを出し分け
- チップラベルに `{stat.withCount}日` を追記

#### A-2. 閾値の強化

```typescript
// 変更前（habitStats.ts）
if (total < 3) continue;

// 変更後: 各グループ最低3件を要求
if (data.withScores.length < 3 || data.withoutScores.length < 3) continue;
```

#### A-3. 損失フレーミング文言

プロスペクト理論：同じ大きさの損失は利益の約2倍の心理的インパクト（Kahneman & Tversky, 1979）。

```
現状: +7点
改善例: 「やめると -7点の可能性」
```

**必須条件：** 「可能性」という留保を入れる。断定表現は医学的・倫理的に不可。
**変更範囲：** `ja.ts` / `en.ts` の i18n キー変更のみ。

---

### 🟡 優先度B — `HabitStat` 型拡張が必要（コスト：中）

#### B-1. 実行率の追加

```typescript
withRate: number;  // withCount / (withCount + withoutCount)
```

表示例: `🏃 運動 週4日ペース（60%）`

#### B-2. 進捗バー（データ蓄積への誘引）

```typescript
confidenceTarget: number;  // 信頼度到達に必要な残り件数
```

表示例:
```
🏃 運動  データ収集中 ████████░░ あと2日で信頼度アップ
```

「データが少なくて表示できない」をポジティブな継続モチベーションに変換。
→ **リテンション（記録継続率）に直接貢献**

---

### 🔴 優先度C — バックエンド変更が必要（コスト：大・保留）

#### C-1. 時系列相関分析

「習慣を始めてから何週でスコアが上がったか」を検出。
週次スコアのローリング平均 × 習慣実行日のオーバーレイが必要。

#### C-2. 多変量解析（交絡対策）

「運動 AND 禁酒」の組み合わせ効果を分離。
Cloud Functions での集計が必要。
