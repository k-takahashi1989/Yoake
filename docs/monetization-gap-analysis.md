## 現状評価（AARRR）

### Acquisition（獲得） — D

- **Android専用**。日本のスマートフォン市場でiOSシェア約65%を切り捨てている
- **Google Play未リリース**（Closed Beta準備中）。有機的ストア流入がゼロ
- シェア・口コミ・招待機能が未実装。バイラル流入経路がない
- ASO（App Store Optimization）戦略の痕跡がない

### Activation（活性化） — C

**強み**
- 5ステップのオンボーディングは適切な構成
- GoalSetupStep で目標を設定させ「あなたの数値に最適化」という認識を植え付けている
- 7日間無料トライアルをオンボーディング最後に訴求（タイミングとして正しい）

**問題点**
- WelcomeStep にアニメーションがない。初回の感動体験が薄い
- 初回データがゼロの状態でスコアリングがなく「今日試せる体験」がない
- Health Connect連携スキップ率が高い場合、スコアが4項目の簡易版になり初日の体験品質が低下

### Retention（継続） — C+

**強み**
- 毎朝AIひとことはリテンションフックとして設計上正しい
- 前日記録なしバナーで記録忘れをリカバリー
- 週次目標ドットで毎日の達成/未達成を可視化

**問題点**
- **ストリーク（連続記録日数）が未実装**。Duolingo等が証明する最強リテンション施策が欠如
- バッジ・実績システムがゼロ。長期継続への報酬設計がない
- ScoreRingにアニメーションがない。毎日最初に見る体験に動きがなく「見に来る理由」が薄い
- AI性格選択（4種類）はリテンション施策として活用されていない

### Revenue（収益化） — B-

**強み**
- ペイウォールは正しい位置（ReportScreen全体）に設置
- 月額¥380はSleep Cycle（¥1,500/月）の1/4で心理的障壁が低い
- 7日間無料トライアルの設計は適切

**問題点**
- **¥380は安すぎる可能性**。競合比で「大したことない機能」という認知バイアスが働くリスク
- **無料版が充実しすぎ**：AIひとこと・睡眠スコア・週次目標ドット・アラーム・Health Connect連携が無料。転換動機が弱い
- レポートペイウォールが「完全ブロック（🔒+リスト）型」。「一部見えてロックされる」体験より転換率が低い
- 年額の節約額訴求が弱い（「月換算¥233」のみで「年間¥1,760お得」という表現がない）

### Referral（紹介） — F

- シェア機能が一切未実装
- 「スコアXX点達成！」「30日連続記録！」のシェア経路がない
- 紹介コード・招待リンクがない
- 口コミが発生する構造的理由がない

## 競合比較と差別化ポイント

| 項目 | Sleep Cycle | AutoSleep | ShutEye | YOAKE |
|---|---|---|---|---|
| 価格 | ¥1,500/月 | ¥610買い切り | ¥1,000/月 | ¥380/月 |
| プラットフォーム | iOS/Android | iOS専用 | iOS/Android | **Android専用** |
| 計測方式 | マイク | Apple Watch | マイク | Health Connect |
| AI分析 | なし | なし | なし | **Claude Haiku** |
| 習慣相関分析 | なし | なし | なし | **あり（有料）** |
| スコアリング | なし | あり | なし | **あり（0-100点）** |

### YOAKEの真の差別化要素

1. **AI × 習慣相関分析**：「カフェインを摂った日はスコアが平均X点低い」という個人最適化された知見を提供できる唯一のアプリ
2. **Health Connectエコシステム**：Androidで最も信頼性が高い睡眠データソース
3. **価格の安さ**：月¥380はコンビニコーヒー1杯分

### 差別化の問題点

- 「AIひとこと」は**機能訴求**になっている。「3週間でスコアが12点改善」という**結果訴求**に変えるべき
- iOS非対応は長期的に致命的。日本市場の65%にアクセス不可
- Health Connect連携の恩恵を初日に体感させる仕組みがない

## 推奨アクションプラン（3フェーズ）

### Phase 1（〜1ヶ月）: リリースと生存基盤の確立

**目標：Google Play本番リリース・最初の有料ユーザー20人獲得**

1. **Cloud Functions デプロイ**（G3）— AIなしでリリースするとレビューが壊滅的になる
2. **Google Play本番リリース**（G1・G2）— HC審査中は手動入力専用版で先行リリース可
3. **ScoreRingアニメーション**（G4）— `strokeDashoffset` を 600ms ease-out + スコアカウントアップ
4. **オンボーディングのフェードアニメーション**（`ui-ux-review.md` Level 1）— 1〜2時間で実装可能
5. **TrialStep スキップボタンの改善**（`color: '#666'` → `#888` 以上・テキストを「後で設定する」に変更）

**KPI：インストール500件・有料転換20件（転換率4%）**

---

### Phase 2（〜3ヶ月）: リテンションと転換率の改善

**目標：MAU 2,000人・有料ユーザー100人**

1. **ストリーク実装**（G5）— ホームに「🔥 23日連続記録中」を表示、3/7/14/30日達成でプッシュ通知
2. **レポートペイウォール改善**（G6）— スコア推移の7日分は見せて以降をぼかす「グレーアウト+CTA」型へ
3. **スコアシェア機能**（G7）— `react-native-view-shot` でキャプチャ → Share API + `#YOAKE睡眠記録` タグ
4. **年額節約額の訴求強化**（G8）— 「月額より年間¥1,760お得」を TrialStep 年額カードに追記
5. **就寝リマインダー通知（有料）**（G9）— 設計書仕様あり。アラームと連携で手動入力精度を向上

**KPI：7日継続率 40%・30日継続率 20%・有料転換率 5%**

---

### Phase 3（〜6ヶ月）: スケールと差別化深化

**目標：MAU 5,000人・有料ユーザー250人・月5万円到達**

1. **バッジ・実績システム**（G10）— 「初めての記録」「7日連続」「スコア90点」等のマイルストーン報酬
2. **iOS展開着手**（G14）— React Nativeベースなので技術的障壁は低い。市場が3倍になる
3. **価格改定**（G12）— ¥500〜¥600/月へ。同じ200ユーザーで月5〜6万円に到達
4. **Androidウィジェット**（G16）— ホーム画面に常時スコア表示でDAU向上
5. **週次AIレポートの画像シェア**（G13）— プレミアム会員のシェアがオーガニック流入を生む

**KPI：月5万円MRR達成・年額比率 40%以上**

## リスクと前提条件

### 技術的リスク

| リスク | 深刻度 | 対策 |
|---|---|---|
| Health Connect申請遅延（1〜4週間） | 高 | 審査前でも手動入力専用でリリース。HC機能はフラグ制御で後から有効化 |
| Cloud Functions APIコスト超過 | 中 | 無料ユーザーのAIひとこと生成コストに上限設定が必要（現在は無制限） |
| Google Play Billing の本番動作 | 高 | TrialStep のIAP実装は正しいが、実機でのE2Eテストが必須 |

### ビジネス的リスク

| リスク | 深刻度 | 対策 |
|---|---|---|
| Android専用による市場制限 | 高 | Phase 3でのiOS展開を計画に組み込む。iOS版なしでは月5万が天井になる可能性 |
| ¥380が価値を下げる認知バイアス | 中 | 価格よりも「AI睡眠コーチ」という価値訴求にシフト。価格はサブテキストで |
| 無料版が充実しすぎて転換しない | 中 | ストリーク可視化・詳細スコア内訳・長期グラフを有料移行の動機として再設計 |
| 個人開発のサポート持続性 | 中 | ユーザー増加に伴うサポート負荷増。FAQ・ヘルプページの整備が必要 |

### 実施前の前提条件

- [ ] Firebase Blazeプラン（従量課金）への移行完了
- [ ] Google Play Developer アカウントの本番アプリ登録完了
- [ ] Google Play Console のサービスアカウントに「財務データ閲覧者」権限付与
- [ ] `functions/.env` の `CLAUDE_API_KEY` 設定（これがないとAI機能が一切動作しない）
- [ ] `firebase deploy --only functions` でCloud Functionsデプロイ完了
